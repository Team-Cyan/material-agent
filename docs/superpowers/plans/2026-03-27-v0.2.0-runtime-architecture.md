# v0.2.0 Runtime Architecture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the pipeline-centered runtime with a session/job architecture that is GUI-ready, multi-backend-ready, and supports resumable file-level execution.

**Architecture:** Introduce `shells -> app -> domain -> ports -> adapters`, add a new runtime schema (`Session`, `Job`, `JobFile`, `Artifact`, `Event`), and progressively route the existing photo workflow through job services until the legacy pipeline path can be retired.

**Tech Stack:** Python 3.14, SQLite, rawpy, OpenCV, ExifTool, httpx, pytest, ruff

---

## File Structure Map

- `src/material_agent/shells/cli/`
  CLI entry and command routing only.
- `src/material_agent/app/`
  Session/job services, executors, runtime orchestration.
- `src/material_agent/domain/`
  Grouping, screening, scoring, commentary, scene, rating rules.
- `src/material_agent/ports/`
  Capability interfaces for models, state, artifacts, metadata, progress.
- `src/material_agent/adapters/`
  OMLX, Ollama, SQLite, RawPy, ExifTool, tqdm, filesystem implementations.
- `tests/`
  New layer-oriented tests plus migration regression coverage.
